**版本:** 1.0  
**作者:** Manus AI  
**日期:** 2026-01-22

---

## 📋 概述

Windows 端有**两个不同的 UI 界面**，它们的定位和功能需要明确区分：

1. **Galaxy Gateway Dashboard** - 系统管理和监控界面
2. **Windows Client UI** - 用户交互和操作界面

---

## 1️⃣ Galaxy Gateway Dashboard

### 🎯 定位

**系统管理员界面** - 用于管理和监控整个 Galaxy 系统的后台管理界面。

### 👥 目标用户

- 系统管理员
- 开发者
- 需要查看系统状态的技术人员

### 🖥️ 访问方式

- **Web 界面**
- 通过浏览器访问: `http://localhost:8000/dashboard`
- 可以从任何设备访问（通过 Tailscale）

### 🎨 UI 风格

- **专业技术风格**
- 深色主题
- 数据密集型界面
- 图表和监控面板

### ⚙️ 核心功能

#### 1. 设备管理
- 查看所有在线设备（Windows、Android、平板等）
- 设备状态监控（在线/离线、CPU、内存、网络）
- 设备注册和注销
- 设备能力查看

#### 2. 任务监控
- 实时任务执行状态
- 任务队列查看
- 任务历史记录
- 任务执行日志

#### 3. API 管理
- API 调用统计
- API 成本分析
- API 配额监控
- API 提供商状态

#### 4. 节点管理
- 79 个功能节点状态
- 节点启动/停止
- 节点日志查看
- 节点配置管理

#### 5. 系统监控
- 系统资源使用情况
- WebSocket 连接状态
- 消息流量统计
- 错误和警告日志

#### 6. 配置管理
- 系统配置编辑
- API Keys 管理
- 节点配置
- 网络配置

## 2️⃣ Windows Client UI

### 🎯 定位

**用户交互界面** - 用户日常使用的主要交互界面，类似 Android 端的灵动岛悬浮窗。

### 👥 目标用户

- 普通用户
- 日常使用者
- 需要快速发送指令的用户

### 🖥️ 访问方式

- **侧边栏界面**
- 按 **Fn 键** 从屏幕右侧滑出
- 始终在后台运行，随时可用

### 🎨 UI 风格

- **极简极客风格**
- 黑白渐变
- CRT 辉光和扫描线效果
- 类似终端的界面
- 与 Android 端灵动岛风格呼应

### ⚙️ 核心功能

#### 1. 自然语言输入
- **语音输入** - 按住说话
- **文本输入** - 快速输入命令
- 输入历史记录
- 智能建议

#### 2. 快速操作
- 常用命令快捷按钮
- 最近使用的操作
- 收藏的操作序列
- 一键执行

#### 3. 状态显示
- 当前任务执行状态
- 系统简要状态
- 通知和提醒
- 设备在线状态（简化版）

#### 4. 设备控制
- 快速切换控制目标设备
- 发送跨设备指令
- 查看其他设备状态

#### 5. 个性化设置
- UI 主题切换
- 快捷键配置
- 语音识别设置
- 通知偏好

## 🔄 两个 UI 的关系

### 协同工作

```
用户 → Windows Client UI → 发送指令 → Galaxy Gateway
                                            ↓
                                    处理和路由任务
                                            ↓
                        ┌───────────────────┼───────────────────┐
                        ↓                   ↓                   ↓
                Windows 自主操纵        Phone A Agent       Phone B Agent
                        ↓                   ↓                   ↓
                    执行任务            执行任务            执行任务
                        ↓                   ↓                   ↓
                    返回结果            返回结果            返回结果
                        └───────────────────┼───────────────────┘
                                            ↓
                                    Galaxy Gateway
                                            ↓
                                    Windows Client UI
                                            ↓
                                        显示结果
```

### 数据流

- **Windows Client UI** → 用户输入 → Galaxy Gateway
- **Galaxy Gateway** → 任务分发 → 各设备 Agent
- **各设备 Agent** → 执行结果 → Galaxy Gateway
- **Galaxy Gateway** → 结果汇总 → Windows Client UI
- **Galaxy Gateway Dashboard** → 监控所有过程

### 使用场景

#### 场景 1: 日常使用
- 用户按 Fn 键呼出 **Windows Client UI**
- 说出指令: "打开微信"
- Windows 自主操纵模块执行
- 结果显示在 Windows Client UI

#### 场景 2: 跨设备操作
- 用户在 **Windows Client UI** 中选择目标设备: Phone A
- 说出指令: "打开微信"
- Galaxy Gateway 路由到 Phone A Agent
- Phone A 执行操作
- 结果返回并显示在 Windows Client UI

#### 场景 3: 系统管理
- 管理员打开浏览器访问 **Galaxy Gateway Dashboard**
- 查看所有设备状态
- 查看 API 成本
- 调整系统配置

## 📝 总结

### Galaxy Gateway Dashboard
- **定位**: 系统管理和监控
- **用户**: 管理员和技术人员
- **访问**: Web 浏览器
- **风格**: 专业技术风格
- **功能**: 全面的系统管理

### Windows Client UI
- **定位**: 用户日常交互
- **用户**: 普通用户
- **访问**: Fn 键侧边栏
- **风格**: 极简极客终端风格
- **功能**: 快速指令输入和执行

### 核心区别

| 特性 | Galaxy Gateway Dashboard | Windows Client UI |
|------|-------------------------|-------------------|
| **定位** | 系统管理 | 用户交互 |
| **访问** | Web 浏览器 | Fn 键侧边栏 |
| **风格** | 专业技术 | 极简极客 |
| **复杂度** | 高 | 低 |
| **使用频率** | 偶尔 | 频繁 |
| **主要功能** | 监控管理 | 发送指令 |

**两个 UI 各司其职，共同构成完整的 Windows 端用户体验。**
